8. how important is the software user guide and the software license?-guys patulong po ​


Answer:

As for the user guide, it’s a better-to-have-it-and-not-need-it-than-to-need-it-and-not-have-it situation. Users don’t read guides much, but when they run into problems, they’re glad there’s some kind of material they can turn to. And every support issue that your guide can resolve, is one less issue you don’t have to deal with yourself. In the long run, time spent on good documentation is time repaid several times over.

A software licence matters to at least define the terms under which the software can be legally used, how it can be copied and/or transferred, and what remedies (if any) are available to the user in case the software doesn’t perform. Even on open source projects that are “do whatever you want”, it’s not uncommon to have a licence explicitly stating that, for the sake of completeness.

Each element within the periodic table has its own block. Within these blocks are identifying numbers and letters. The number on the very top is called the atomic number and it represents the number of protons in a single atom of the element. For the element helium (shown above) the atomic number is two. The letters in each block represent the atomic symbol. This is essentially an abbreviation of the elements name. The atomic symbol for helium (shown above) is He. What is Software Licensing?  

The software license refers to the ways and terms which the defined software can be used in a legal way. Such way includes how the software can be distributed, copied and transferred. There are also solutions enumerated in any case that the software doesn't performed it's intended function.  

What is Software User Guide?  

The software user guide is a step-by-step procedure and guide on how to set-up the software, it's requirements, it's installation, and how to use it. The user guide also contains basic troubleshooting tips like what the user can do when they encountered some problems.
